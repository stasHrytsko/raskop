# Раскоп — push-your-luck раскопки

Мини-игра в духе «Сапёра» с механикой жадности: копай клетки за добычей и реши,
когда вовремя уйти. Поле 6×6, под клетками ловушки 💀 и добыча 🪙🧰💎. Цифра в
открытой клетке — число ловушек среди 8 соседей (с диагоналями).

## Отступление от корневого стандарта

Корневой `../CLAUDE.md` требует **один файл `index.html`**. Эта игра намеренно
**разбита на логические файлы** (по просьбе для удобства поддержки). Остальные
требования стандарта в силе: mobile-first (max-width 430px), тёмная тема через
переменные в `:root`, без внешних зависимостей.

## Структура файлов

```
index.html        — разметка + подключение css/js (классические <script>, по порядку)
css/styles.css    — все стили и :root-переменные тёмной темы
js/config.js      — игровые константы + DOM-хелпер $()
js/field.js       — генерация поля, соседи, подсчёт ловушек, множитель
js/ui.js          — попапы/оверлеи (интро, конец захода, уровень, гейм-овер)
js/render.js      — отрисовка поля и HUD, анимация добычи
js/game.js        — состояние S и игровой цикл (start/nextDive/tap/leave/endDive) + запуск
concept.txt       — дизайн-документ и обоснование механик
```

### Как связаны модули
- Это **классические скрипты** (не ES-модули) — общий глобальный scope, работают и
  по `file://`, и на GitHub Pages. **Порядок подключения важен**: config → field → ui → render → game.
- Общее состояние — глобальная `S` (объявлена в `js/game.js`).
- Менять порядок `<script>` в `index.html` нельзя: `ui`/`render` обращаются к функциям,
  объявленным в `game.js`, но вызывают их только в рантайме после `start()`.

## Баланс (крутить здесь)

Весь баланс — в [js/config.js](js/config.js):
`TRAPS` (ловушек), `QUOTA0`/`QUOTA_STEP` (цель и её рост), `DIVES` (заходов),
`TH2`/`TH3` (вскрытий для множителей ×2/×3).

## Правила игры
- Первое вскрытие за заход всегда безопасно.
- Наступил на ловушку — рюкзак сгорает, заход потерян (накопленный прогресс остаётся).
- «Уйти» в любой момент: `рюкзак × множитель` уходит в прогресс уровня.
- Множитель растёт от числа вскрытий за заход: ≥5 → ×2, ≥8 → ×3.
- Цель уровня за `DIVES` заходов; излишек переносится на следующий уровень.
