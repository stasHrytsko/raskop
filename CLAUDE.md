# Раскоп — push-your-luck раскопки

Соло-игра в духе «Сапёра» с механикой жадности (полная спецификация — [concept.txt](concept.txt)).
Поле 6×6, на нём **4 монеты** и **10 ловушек**. Цифра в открытой клетке — число ловушек
среди 8 соседей (с диагоналями). Копай за монетами и реши, когда вовремя уйти.

Параметры движка выверены Монте-Карло (бот ≈ сильный игрок, 6000 симуляций) — таблица
ожидаемого выхлопа по стратегиям и обоснование в шапке [js/config.js](js/config.js).

## Модель: забег

Игра — это **забег из 9 уровней подряд** (не свободный выбор уровня). Уровень = до 3 заходов.
Провал (ловушка без оставшихся заходов или невзятая квота за 3 захода) завершает **весь
забег**; показывается экран результата с **рекордом** (хранится в `localStorage`, ключ `raskop_best`).

## Страницы (3, по корневому стандарту)

Одна `.screen` активна за раз, переключение — `showScreen()` в [js/nav.js](js/nav.js):

1. **Правила** (`#screen-rules`) — название, правила, кнопка «Играть».
2. **Карта забега** (`#screen-levels`) — сетка **9 уровней 3×3** (1–3 Легко, 4–6 Средне,
   7–9 Сложно) как карта прогресса: пройдено ✓ / текущий / закрыто. Рекорд + кнопки
   «Начать забег» / «Продолжить» / «Начать заново». Карта **не кликабельна** (забег линейный).
3. **Игра** (`#screen-game`) — хедер «← УР. N» (на карту) + «Заход N/3», поле, рюкзак, кнопка «Уйти».

Попапы (оверлей `#ovl`, в [js/ui.js](js/ui.js)): `showDiveEnd` (между заходами),
`showLevelClear` (уровень пройден → следующий), `showRunWin` (все 9 пройдены),
`showRunOver` (забег окончен + причина + рекорд).

## Ключевые правила (из concept.txt)

- Первое вскрытие захода — честный старт: **0 ловушек среди соседей** (`FIRST_TAP_BLANK`,
  `clearStart()` переносит ловушки из зоны старта). Не просто «не ловушка», а цифра 0.
- **Ловушка → весь прогресс уровня сгорает** (`S.prog = 0`, включая монеты прошлых заходов),
  поле полностью раскрывается. Заход потерян; если заходы ещё есть — продолжаем с нуля.
- «Уйти»: `рюкзак × множитель` → прогресс уровня. С пустым рюкзаком уйти нельзя.
- Все 4 монеты собраны → заход завершается автоматически (безопасно).
- Множитель за заход: ≥8 безопасных вскрытий → ×2, ≥14 → ×3, ≥20 → ×4 (`TH2`/`TH3`/`TH4`).
  Сбрасывается каждый заход. Свободная зона при 10 ловушках ≈14 клеток — ×4 уже за риском.
- Излишек монет на следующий уровень не переносится.
- Игроку **не** показываем: число монет/ловушек на поле, счётчик открытых клеток, флажки.

## Отступление от стандарта

Корневой `../CLAUDE.md` требует **один файл `index.html`**. Игра намеренно **разбита на
файлы** для поддержки. Остальное по стандарту: mobile-first (max-width 430px), тёмная тема
через `:root`, без внешних зависимостей.

## Структура файлов

```
index.html        — 3 секции-страницы + оверлей, подключение css/js по порядку
css/styles.css    — стили, :root-переменные, система .screen / карта забега
js/config.js      — константы баланса + quotaFor()/tierFor() + DOM-хелпер $()
js/field.js       — генерация поля newField() с балансом, соседи, множитель
js/ui.js          — попапы: showDiveEnd / showLevelClear / showRunWin / showRunOver
js/render.js      — отрисовка поля и HUD, анимация монеты
js/nav.js         — showScreen(), buildMap() (карта забега), gotoMap(), initNav()
js/game.js        — состояние (best/run/S) и цикл + старт (loadBest + initNav)
concept.txt       — дизайн-документ (источник правды по механикам)
```

### Как связаны модули
- **Классические скрипты** (не ES-модули) — общий глобальный scope, работают по `file://`
  и на GitHub Pages. **Порядок важен**: config → field → ui → render → nav → game. Менять нельзя.
- Глобалы в `js/game.js`: `best` (рекорд), `run` (`{level}` или null), `S` (заход).
- Точка входа — конец `js/game.js`: `loadBest(); initNav();`.

## Баланс (крутить здесь)

Весь баланс — в [js/config.js](js/config.js): `COINS`, `TRAPS`, `QUOTA_START`/`QUOTA_STEP`,
`DIVES`, `TH2`/`TH3`/`TH4`, `MAX_CLUE`, `FIRST_TAP_BLANK`. Генератор поля гарантирует: цифры
только 1..`MAX_CLUE` (нет безопасной клетки с >3 ловушками рядом) и ≥2 монеты с нулевым
риском; иначе перегенерация (см. `newField`).

> **Параметры выверены Монте-Карло** (таблица и обоснование — в шапке `config.js`).
> Ключевые отступления от исходного concept:
> - `TRAPS = 10`, фиксировано для всех уровней (было растущее `TRAPS_BY_LEVEL`/`TRAPS=4`).
>   При 4 ловушках нет спреда стратегий — уровень закрывался за один безрисковый заход.
>   Сложность теперь растёт **только квотой** (`quotaFor`), число ловушек постоянно.
> - Пороги множителя подняты до `8/14/20` (было `6/9`) и добавлен тир ×4.
> - Безусловный EV почти ровный по стратегиям (8.3/8.5/8.8) — жадность покупает дисперсию,
>   не очки: ×2–×3 берутся даром, ×4 и последние монеты — только за риском.
>
> Желательная раскладка «2 лёгкие / 1 средняя / 1 рисковая монета» (concept §«Баланс
> генерации») реализована частично: гарантированы только ≥2 монеты нулевого риска.
