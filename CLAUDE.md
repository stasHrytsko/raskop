# Раскоп — push-your-luck раскопки

Мини-игра в духе «Сапёра» с механикой жадности: копай клетки за добычей и реши,
когда вовремя уйти. Поле 6×6, под клетками ловушки 💀 и добыча 🪙🧰💎. Цифра в
открытой клетке — число ловушек среди 8 соседей (с диагоналями).

## Структура страниц (по корневому стандарту)

Игра следует корневому `../CLAUDE.md` — **3 страницы** (одна `.screen` активна за раз,
переключение через `showScreen()` в [js/nav.js](js/nav.js)):

1. **Правила** (`#screen-rules`) — название, описание, список правил, кнопка «Выбрать уровень».
2. **Выбор уровня** (`#screen-levels`) — сетка **9 уровней 3×3**: 1–3 Легко (зелёные),
   4–6 Средне (янтарные), 7–9 Сложно (красные). Кнопка «← Назад».
3. **Игра** (`#screen-game`) — хедер «← УР. N» (возврат к выбору) + счётчики, поле, управление.

Попапы (оверлей `#ovl`): **победа** (`showWin` — «Поздравляю, прошёл!», кнопки
«Уровень N+1 →» / «Ещё раз» / «К уровням»), **поражение** (`showLose` — «Попробовать
снова» / «К уровням»), плюс промежуточный **конец захода** (`showDiveEnd`).

## Отступление от стандарта

Корневой стандарт требует **один файл `index.html`**. Эта игра намеренно
**разбита на логические файлы** для удобства поддержки. Остальное по стандарту:
mobile-first (max-width 430px), тёмная тема через `:root`, без внешних зависимостей.

## Структура файлов

```
index.html        — 3 секции-страницы + оверлей, подключение css/js по порядку
css/styles.css    — стили, :root-переменные, система .screen/.screen.active
js/config.js      — константы + массив LEVELS (9 уровней) + DOM-хелпер $()
js/field.js       — генерация поля newField(traps), соседи, ловушки, множитель
js/ui.js          — попапы: showDiveEnd / showWin / showLose
js/render.js      — отрисовка поля и HUD, анимация добычи
js/nav.js         — showScreen(), buildLevels() (сетка 3×3), initNav()
js/game.js        — состояние S и цикл (startLevel/nextDive/tap/leave/endDive) + initNav()
concept.txt       — дизайн-документ и обоснование механик
```

### Как связаны модули
- **Классические скрипты** (не ES-модули) — общий глобальный scope, работают и по
  `file://`, и на GitHub Pages. **Порядок подключения важен**: config → field → ui →
  render → nav → game. Менять нельзя.
- Общее состояние — глобальная `S` (объявлена в `js/game.js`).
- Точка входа — `initNav()` в конце `js/game.js`: строит сетку уровней и показывает страницу правил.
- Уровень индексируется с 0 (`S.level`), на экране показывается `S.level + 1`.

## Баланс (крутить здесь)

Весь баланс — в [js/config.js](js/config.js): массив `LEVELS` (по уровню: `traps`,
`quota`, `dives`) и пороги множителей `TH2`/`TH3`. Добыча на поле фиксирована
(6×🪙=2, 3×🧰=4, 1×💎=7 → 31).

> Баланс уровней предварительный — выставлен «на глаз», стоит прогнать и подкрутить.

## Правила игры
- Первое вскрытие за заход всегда безопасно.
- Наступил на ловушку — рюкзак сгорает, заход потерян.
- «Уйти» в любой момент: `рюкзак × множитель` уходит в прогресс уровня.
- Множитель растёт от числа вскрытий за заход: ≥5 → ×2, ≥8 → ×3.
- Цель уровня надо взять за `dives` заходов; уровни независимы (прогресс не переносится).
